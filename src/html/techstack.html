Kunskap

HTML5

HTML är ett markup-språk som används för att strukturera innehåll på webbsidor. Det står för Hyper Text Markup Language och använder taggar och attribut för att definiera olika delar av en webbsida, till exempel rubriker, stycken, bilder och länkar. HTML är en grundläggande teknik inom webbutveckling och används ofta tillsammans med CSS och JavaScript för att skapa interaktiva och responsiva webbsidor och applikationer. HTML-koden är läsbar av webbläsare och används för att bygga en grundläggande struktur för webbsidan, vilket gör det möjligt för utvecklare att lägga till mer komplex funktionalitet och design.

CSS3

CSS (Cascading Style Sheets) är en stilskapande teknik som används för att formatera och designa webbsidor. CSS används för att definiera visuella egenskaper på webbsidan, såsom färger, typsnitt, layout och animationer. Genom att separera design från innehåll kan webbutvecklare enkelt ändra webbsidans utseende genom att justera CSS-koden utan att ändra HTML-koden. CSS är en viktig del av webbutveckling och används tillsammans med HTML och JavaScript för att skapa interaktiva och anpassade webbsidor och applikationer som fungerar på olika enheter och plattformar.

SASS

Sass (Syntactically Awesome Style Sheets) är en preprocessor för CSS som erbjuder fler funktioner än traditionell CSS. Sass gör det möjligt att skriva CSS-kod med variabler, nesting, mixins och funktioner, vilket gör det lättare att skapa och underhålla CSS-kod. Sass-koden kompileras sedan till vanlig CSS-kod som kan användas på webbsidor och applikationer. Sass är en populär teknik inom webbutveckling och används tillsammans med HTML och JavaScript för att skapa anpassade och responsiva webbsidor och applikationer.

JavaScript

JavaScript är ett högnivåprogrammeringsspråk som används för att skapa interaktiva och dynamiska webbsidor och applikationer. Det är ett klientsidespråk som körs direkt i webbläsaren och kan användas för att manipulera HTML och CSS på sidan, samt hantera användarinteraktioner. JavaScript är också ett populärt server-side-språk som kan användas tillsammans med Node.js för att bygga skalbara och realtidsapplikationer. Eftersom det är så vanligt förekommande är det viktigt för alla front-end-utvecklare att ha en god förståelse för JavaScript och dess ramverk och bibliotek.

Typescript

TypeScript är ett programmeringsspråk som är baserat på JavaScript, men som har utvecklats för att göra det lättare att bygga stora och komplexa applikationer. Det är ett typat språk, vilket innebär att det ger utvecklaren möjlighet att definiera variabler och funktioner med specifika datatyper. Detta kan hjälpa till att minska antalet fel som kan uppstå i större projekt. TypeScript har också ett kraftfullt typsystem och stöd för klasser och gränssnitt, vilket kan göra det enklare att underhålla och utveckla större projekt. Eftersom det är baserat på JavaScript, är det också lätt att integrera med befintliga JavaScript-projekt.

React

React är ett JavaScript-baserad bibliotek för att skapa användargränssnitt på webbsidor och applikationer. React använder sig av en komponentbaserad struktur där olika delar av webbsidan eller applikationen representeras som fristående komponenter. Dessa komponenter kan sedan sättas samman för att bygga upp en komplett användargränssnitt. React är utvecklat av Facebook och är ett populärt verktyg inom webbutveckling, som erbjuder hög prestanda och enkel hantering av dataflöde och tillstånd. React används ofta tillsammans med andra tekniker som Redux, TypeScript och CSS-preprocessor för att skapa avancerade webbsidor och applikationer.

Node.js

Node.js är en plattform som gör det möjligt att köra JavaScript-kod utanför en webbläsare. Node.js används för att skapa skalbara och högpresterande applikationer på serversidan och är baserad på Googles Chrome V8-motor. Node.js erbjuder enkel hantering av asynkrona händelser, vilket gör det lätt att bygga skalbara webbapplikationer som kan hantera många samtidiga anslutningar. Node.js har ett stort ekosystem av moduler och verktyg som gör det lätt att utveckla och underhålla applikationer. Det är ett populärt verktyg inom webbutveckling som används av många stora företag och organisationer.

Express.js

Express.js är en flexibel och minimalistisk webbramverk för Node.js-plattformen som används för att bygga skalbara webbapplikationer. Express.js är baserat på Node.js och erbjuder ett enkelt och kraftfullt sätt att hantera HTTP-förfrågningar och svar, routing och mellanlagring. Express.js har ett stort urval av plugins och middleware som gör det lätt att utöka dess funktionalitet och anpassa applikationen efter specifika behov. Det är ett populärt verktyg inom webbutveckling och används för att bygga allt från enkla webbapplikationer till komplexa API:er och mikrotjänster.

Cypress

Cypress är ett ramverk för front-end testning av webbapplikationer. Cypress har enkla och lättförståeliga API för att skriva testfall och kan användas för att testa både användarinteraktioner och API-anrop. Cypress kör testfallen i en automatisk webbläsare och ger feedback på resultatet i realtid. Cypress är ett kraftfullt verktyg för att säkerställa att en webbapplikation fungerar som avsett och ger utvecklare möjlighet att upptäcka buggar och problem tidigt i utvecklingsprocessen. Cypress är enkelt att integrera med andra verktyg och system och är ett populärt val för många webbutvecklare och företag.

Vitest

Vitest är ett verktyg för automatisk testning av webbapplikationer som bygger på WebDriver-protokollet. Vitest är en del av verktygssamlingen Selenium och gör det möjligt att skapa och köra automatiserade tester på olika webbläsare och plattformar. Vitest stöder olika programmeringsspråk som Java, Python, C# och Ruby. Vitest kan användas för att testa olika typer av användargränssnitt, som HTML, CSS och JavaScript. Det är ett användbart verktyg för webbutvecklare som vill säkerställa kvaliteten och stabiliteten i sina webbapplikationer genom att automatisera testprocessen.

End-to-end

End-to-end testning är en metod inom mjukvarutestning som testar en applikation från början till slut för att säkerställa att applikationen fungerar som avsett i en verklig miljö. End-to-end testning fokuserar på att testa alla funktioner och interaktioner i applikationen, från gränssnittet till databasåtkomst och bakgrundsuppgifter. Detta säkerställer att applikationen fungerar som en helhet och att alla delar är integrerade korrekt. End-to-end testning är en viktig del av kvalitetssäkring och används ofta inom webbutveckling för att upptäcka problem tidigt och säkerställa att en applikation är tillförlitlig och fungerar korrekt.

Enhets tester

Unit testing är en metod inom mjukvarutestning som fokuserar på att testa individuella enheter av en applikation, såsom en funktion, en klass eller en modul, isolerat från resten av applikationen. Syftet med unit testing är att säkerställa att varje enhet fungerar som avsett och att alla avvikande beteenden upptäcks tidigt i utvecklingsprocessen. Unit testing används ofta för att säkerställa kodkvalitet och underlättar felsökning genom att isolera och identifiera problematisk kod. Genom att använda unit testing kan utvecklare säkerställa att varje enhet av koden fungerar som avsett, vilket resulterar i mer robust och pålitlig kod.

Integrations tester

Integration testing är en metod inom mjukvarutestning som fokuserar på att testa hur olika delar av en applikation fungerar tillsammans, efter att de har testats individuellt genom unit testing. Syftet med integration testing är att upptäcka problem som kan uppstå när flera enheter samverkar och för att säkerställa att de fungerar korrekt i en integrerad miljö. Integration testing säkerställer också att olika delar av applikationen kommunicerar med varandra på ett korrekt sätt och att de hanterar och överför data som förväntat. Integration testing är en viktig del av kvalitetssäkring och används ofta för att upptäcka problem som kan uppstå när olika delar av en applikation integreras.

Databaser

Databaser är en viktig komponent inom webbutveckling och används för att lagra och hantera stora mängder data på ett organiserat sätt. En databas består av en samling av tabeller, där varje tabell innehåller information om en specifik typ av data. Databaser kan användas för att lagra allt från användarinformation till produktdata och transaktionshistorik. Det finns olika typer av databaser som relationella databaser, NoSQL-databaser och grafiska databaser. Valet av databastyp beror på applikationens behov och krav på datahantering. Databaser kan integreras med webbapplikationer med hjälp av databasdrivrutiner och SQL-språket för att hämta och manipulera data.

MySQL

MySQL är en relationell databas som används för att lagra och hantera data för webbapplikationer. MySQL är baserat på SQL-språket och använder rader och kolumner för att lagra data i olika tabeller. MySQL är skalbar och kan hantera stora mängder data, vilket gör det populärt för webbapplikationer med höga krav på prestanda och skalbarhet. MySQL är också en av de mest populära databaserna inom webbutveckling och används av många stora webbplatser. MySQL kan integreras med olika programvaruramverk och plattformar, vilket gör det till en allsidig och flexibel databas för webbutveckling.

MongoDB

MongoDB är en NoSQL-databas som används för att lagra och hantera data för webbapplikationer. MongoDB är baserad på dokumentorienterad databashantering och använder JSON-formatet för att lagra data i dokument istället för traditionella rader och kolumner. Detta gör det lättare att hantera data som har en komplex struktur och gör det möjligt att söka efter data på ett mer flexibelt sätt. MongoDB är skalbar och kan hantera stora mängder data, vilket gör det populärt för webbapplikationer med höga krav på prestanda och skalbarhet. MongoDB kan integreras med olika programvaruramverk och plattformar, vilket gör det till en allsidig och flexibel databas för webbutveckling.

Prisma

Prisma är ett modern ORM-verktyg (Object-Relational Mapping) för databasåtkomst som används inom webbutveckling. Prisma är baserat på GraphQL och genererar automatiskt en typsäker databasåtkomst API från en data modell som definieras av utvecklaren. Prisma kan integreras med olika databaser som PostgreSQL, MySQL och MongoDB. Med Prisma kan utvecklare lättare hantera databasåtkomst och hantera förändringar i databas scheman. Prisma ger också avancerade funktioner som prestandaoptimering och säkerhetshantering för databasåtkomst. Prisma är ett kraftfullt verktyg för att underlätta databasåtkomst för webbutvecklare.

REST API

REST API står för Representational State Transfer API och är en arkitekturstil för webbtjänster. RESTful API är byggd på HTTP-protokollet och är en vanlig metod för att skapa API:er som kan användas av andra applikationer. RESTful API använder HTTP-metoder som GET, POST, PUT och DELETE för att interagera med resurser och skicka data mellan klient och server. RESTful API är lätta att använda och skala, eftersom de är byggda på standardiserade protokoll och format. De är också självständiga och inte bundna till en specifik teknologi eller plattform, vilket gör dem till en flexibel och användbar lösning för att bygga applikationer.

Websockets

Websockets är en teknik för tvåvägs kommunikation mellan klienter och servrar i realtid. Istället för att använda det traditionella HTTP-protokollet, som kräver att en klient begär data från en server och servern svarar, öppnar websockets en permanent anslutning mellan klienten och servern. Detta möjliggör att data kan skickas i realtid från servern till klienten och vice versa, utan att klienten behöver begära data varje gång. Websockets är användbart för applikationer som kräver realtidsuppdateringar, som chattapplikationer, onlinespel och finansiella applikationer. Websockets kan integreras i webbapplikationer med hjälp av API:er som WebSocket API i JavaScript.

Git

Git är ett öppet versionshanteringssystem som används för att hantera källkod för programvara. Det utvecklades av Linus Torvalds 2005 och har sedan dess blivit ett av de mest populära verktygen för versionshantering. Git tillåter utvecklare att spåra ändringar i källkod, samarbeta med andra utvecklare och hantera kodprojekt. Det använder sig av en distribuerad arkitektur, vilket innebär att alla utvecklare har en egen kopia av källkoden och kan bidra till projektet utan att påverka andras arbete. Git är också kraftfullt och anpassningsbart med många olika funktioner som grenar, sammanfogning och återställning av tidigare versioner av koden.

Node Package Manager

NPM, eller Node Package Manager, är en pakethanterare för JavaScript som används för att ladda ner och hantera tredjepartspaket och bibliotek. Det är standardverktyget för att hantera paket i Node.js-miljön och ger enkel åtkomst till tusentals fria och öppna källkodspaket. Utvecklare kan använda NPM för att spara tid och undvika att skriva ny kod genom att återanvända befintliga paket som innehåller funktioner, verktyg och annan funktionalitet. Paketen kan enkelt installeras, uppdateras och hanteras via terminalen eller NPM:s webbgränssnitt. NPM kan också användas för att publicera egna paket som andra utvecklare kan använda och bidra till.
